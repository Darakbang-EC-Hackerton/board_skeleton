# 사용법
1. 해당 페이지의 코드를 zip로 다운로드받고 작업 환경에서 압축해제 하세요.
2. 코드 편집기(인텔리제이)에서 압축해제한 파일을 열고 요구 사항을 구현해 주세요.


# 프로젝트 구조
가장 간단하고 흔히 쓰이는 레이어드 아키텍쳐로 파일구조를 구성하였습니다.
![image](https://github.com/user-attachments/assets/d5c62546-5bf5-4921-9fc0-ca2c90f5edb1)

이 중 구현해야 할 계층은 크게 세 가지입니다.
- DTO, 엔티티 계층의 코드 리팩토링
- 서비스 계층의 메서드 구현
- ApiController의 추가 기능 구현


# 요구 사항
---
## 기본적인 요구 사항
### 코드 리팩토링
>롬복의 사용방법 가이드라인을 참고하여 DTO와 엔티티 파일을 리팩토링하세요.


### 서비스 계층 구현
>서비스 계층의 비어있는 메서드들을 구현해 주세요. 기본적인 api의 요구 사항은 다음과 같습니다.
- [ ] GET "/api/articles" : 모든 게시글 조회 api
- [ ] GET "/api/articles/{id}" : 게시글 단건 조회 api
- [ ] POST "/api/articles" : 게시글 생성 api
- [ ] PATCH "/api/articles/{id}" : 게시글 수정 api
- [ ] DELETE "/api/articles/{id}" : 게시글 삭제 api


## 추가적인 요구 사항
### 예외 설정
>각 api에 대해서 잘못된 요청을 보낸 경우 적절한 예외 설정을 해주세요.

ResponseEntity를 사용하여 오류 메시지를 HTTP를 통해 응답하도록 합니다.

### 제목, 내용으로 게시글 검색하기
>컨트롤러에서 추가 기능을 구현해 주세요


### 기타 기능
>위의 기능 이외에도 기타 기능을 구현해도 됩니다. 추가 기능을 많을수록 평가 점수에 긍정적인 요소입니다.

이를테면 특정 제목이 들어간 게시글 모두 삭제와 같이 구체적인 기능을 구현해도 좋습니다.
